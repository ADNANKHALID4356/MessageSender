# 🔑 Required Credentials — MessageSender VPS Deployment

> **Security notice:** Never share credentials with anyone, including AI assistants or agents.
> All secrets go directly onto your VPS or into your GitHub repository's Secrets settings — by you, the owner.

This document is the single source of truth for every credential needed to deploy
MessageSender to a VPS. It tells you:
- What each credential is and why it's needed
- How to generate or obtain it
- Exactly where to put it

---

## Overview — Two Places Credentials Live

| Location | Used for | How to set |
|---|---|---|
| **`.env.prod`** on the VPS | Docker Compose reads these at runtime | Create from `.env.prod.example`, fill values, `chmod 600 .env.prod` |
| **GitHub Repository Secrets** | Auto-deploy workflow SSHes to your VPS | GitHub → Settings → Secrets → Actions → New secret |

---

## Part A — GitHub Repository Secrets

These are needed for the automated deploy workflow (`.github/workflows/deploy.yml`) to SSH into
your VPS and deploy on every push to `main`.

Navigate to: **GitHub → Your Fork → Settings → Secrets and variables → Actions → New repository secret**

---

### A1. `VPS_HOST` ✅ Required

| Field | Value |
|---|---|
| **What it is** | The IP address or hostname of your VPS |
| **Example** | `203.0.113.42` or `vps1.yourdomain.com` |
| **How to get it** | Your VPS provider's dashboard (DigitalOcean, Hetzner, Vultr, etc.) shows the public IP of your Droplet/instance |
| **Where to put it** | GitHub Secret named `VPS_HOST` |

---

### A2. `VPS_USER` ✅ Required

| Field | Value |
|---|---|
| **What it is** | The SSH username on the VPS that the deploy workflow will log in as |
| **Recommended value** | `deploy` (a non-root user created by `vps-setup-ubuntu.sh`) |
| **How to get it** | You define this when you run `sudo bash scripts/vps-setup-ubuntu.sh [username]` |
| **Where to put it** | GitHub Secret named `VPS_USER` |

---

### A3. `VPS_SSH_KEY` ✅ Required

| Field | Value |
|---|---|
| **What it is** | The **private** SSH key that the deploy workflow uses to connect to your VPS |
| **How to generate** | See below |
| **Where to put it** | GitHub Secret named `VPS_SSH_KEY` (paste the entire private key contents) |

**Generating a dedicated deploy SSH key pair** (run on your local machine):

```bash
# Generate a dedicated key for CI/CD (do NOT use your personal key)
ssh-keygen -t ed25519 -C "messagesender-deploy" -f ~/.ssh/messagesender_deploy

# This creates:
#   ~/.ssh/messagesender_deploy      ← PRIVATE key → paste into GitHub Secret VPS_SSH_KEY
#   ~/.ssh/messagesender_deploy.pub  ← PUBLIC key  → add to your VPS
```

**Add the public key to your VPS:**

```bash
# On your local machine
ssh-copy-id -i ~/.ssh/messagesender_deploy.pub deploy@YOUR_VPS_IP

# Or manually: paste the .pub contents into this file on the VPS:
#   /home/deploy/.ssh/authorized_keys
```

**Add the private key to GitHub Secrets:**

```bash
# Print the private key — copy every line including -----BEGIN/END-----
cat ~/.ssh/messagesender_deploy
```

Paste the entire output (including `-----BEGIN OPENSSH PRIVATE KEY-----` header and footer) as the value for the `VPS_SSH_KEY` GitHub Secret.

---

### A4. `VPS_PORT` ⚡ Optional

| Field | Value |
|---|---|
| **What it is** | SSH port on your VPS |
| **Default** | `22` (no need to set this unless you changed it) |
| **Where to put it** | GitHub Secret named `VPS_PORT` |

---

### A5. `VPS_APP_DIR` ⚡ Optional

| Field | Value |
|---|---|
| **What it is** | Directory where the app lives on the VPS |
| **Default** | `/opt/messagesender` (no need to set this unless you want a different path) |
| **Where to put it** | GitHub Secret named `VPS_APP_DIR` |

---

### A6. `FRONTEND_URL` ⚡ Optional (for deploy status URL)

| Field | Value |
|---|---|
| **What it is** | The public URL of your frontend — shown as the deployment URL in GitHub Actions |
| **Example** | `https://app.yourdomain.com` |
| **Where to put it** | GitHub Secret named `FRONTEND_URL` |

## Part B — `.env.prod` File on the VPS

These are the application secrets loaded by Docker Compose at runtime.
They live in `/opt/messagesender/.env.prod` on your VPS — **never in git**.

**Quickest way to create `.env.prod`:**

```bash
# On your VPS, from the app directory:
bash scripts/generate-secrets.sh --write

# This auto-fills all generated secrets.
# You only need to manually edit the items marked 🖊️ below.
```

Then edit the remaining fields:

```bash
vim /opt/messagesender/.env.prod
```

---

### B1. `POSTGRES_PASSWORD` ✅ Required — Auto-generated

| Field | Value |
|---|---|
| **What it is** | Password for the PostgreSQL database |
| **Security level** | Strong random — minimum 32 characters |
| **Auto-generated by** | `bash scripts/generate-secrets.sh --write` |
| **Or generate manually** | `openssl rand -base64 32` |
| **⚠️ Warning** | Never change this after first deployment without migrating the database |

---

### B2. `REDIS_PASSWORD` ✅ Required — Auto-generated

| Field | Value |
|---|---|
| **What it is** | Password for the Redis cache/queue |
| **Auto-generated by** | `bash scripts/generate-secrets.sh --write` |
| **Or generate manually** | `openssl rand -base64 32` |

---

### B3. `JWT_SECRET` ✅ Required — Auto-generated

| Field | Value |
|---|---|
| **What it is** | Secret used to sign JWT access tokens (user sessions) |
| **Security level** | Minimum 128 hex characters (64 bytes) |
| **Auto-generated by** | `bash scripts/generate-secrets.sh --write` |
| **Or generate manually** | `openssl rand -hex 64` |
| **⚠️ Warning** | If you change this, all logged-in users will be logged out |

---

### B4. `JWT_REFRESH_SECRET` ✅ Required — Auto-generated

| Field | Value |
|---|---|
| **What it is** | Secret used to sign JWT refresh tokens (must differ from `JWT_SECRET`) |
| **Auto-generated by** | `bash scripts/generate-secrets.sh --write` |
| **Or generate manually** | `openssl rand -hex 64` |

---

### B5. `ENCRYPTION_KEY` ✅ Required — Auto-generated

| Field | Value |
|---|---|
| **What it is** | AES-256 key used to encrypt Facebook access tokens in the database |
| **Security level** | Exactly 64 hex characters (32 bytes) |
| **Auto-generated by** | `bash scripts/generate-secrets.sh --write` |
| **Or generate manually** | `openssl rand -hex 32` |
| **🚨 Critical warning** | **NEVER change this after first deployment.** All stored Facebook tokens become unreadable. Back this up in a password manager. |

---

### B6. `FRONTEND_URL` ✅ Required — 🖊️ You provide this

| Field | Value |
|---|---|
| **What it is** | Public URL of the frontend (used for CORS and email links) |
| **Format** | Full URL, no trailing slash |
| **Example** | `https://app.yourdomain.com` |
| **How to get it** | Your domain name from your domain registrar. Point an A record to your VPS IP. |

---

### B7. `NEXT_PUBLIC_API_URL` ✅ Required — 🖊️ You provide this

| Field | Value |
|---|---|
| **What it is** | URL the browser uses to reach the backend API (baked into the frontend bundle) |
| **Format** | Full URL including `/api/v1`, no trailing slash |
| **Example** | `https://api.yourdomain.com/api/v1` or `https://yourdomain.com/api/v1` |

---

### B8. `NEXT_PUBLIC_SOCKET_URL` ✅ Required — 🖊️ You provide this

| Field | Value |
|---|---|
| **What it is** | URL the browser uses for WebSocket connections (real-time updates) |
| **Format** | Full URL without path, no trailing slash |
| **Example** | `https://api.yourdomain.com` |
| **Note** | Usually the same as `NEXT_PUBLIC_API_URL` but without `/api/v1` |

---

### B9. `FACEBOOK_APP_ID` ✅ Required — 🖊️ You provide this

| Field | Value |
|---|---|
| **What it is** | Your Facebook App's numeric ID |
| **How to get it** | [Facebook Developer Console](https://developers.facebook.com/apps) → Select your app → App ID shown at the top |
| **Prerequisite** | You must have a Facebook Developer account and a Facebook App |

**How to create a Facebook App (if you don't have one):**

1. Go to [developers.facebook.com/apps](https://developers.facebook.com/apps)
2. Click **Create App**
3. Choose **Business** type
4. Fill in app name (e.g. "MessageSender") and contact email
5. After creation, the App ID is shown in the dashboard

---

### B10. `FACEBOOK_APP_SECRET` ✅ Required — 🖊️ You provide this

| Field | Value |
|---|---|
| **What it is** | Your Facebook App's secret key (used to verify webhook signatures) |
| **How to get it** | Facebook Developer Console → Your App → Settings → Basic → **App Secret** (click Show) |
| **⚠️ Warning** | Treat like a password — never share or commit to git |

---

### B11. `FACEBOOK_WEBHOOK_VERIFY_TOKEN` ✅ Required — Auto-generated

| Field | Value |
|---|---|
| **What it is** | A random token **you define** that Facebook sends back to verify your webhook |
| **Auto-generated by** | `bash scripts/generate-secrets.sh --write` |
| **Or generate manually** | `openssl rand -hex 16` |
| **Where also used** | Must be entered in Facebook Developer Console → Webhooks → Verify Token field |

**After deploying, configure the webhook in Facebook:**

1. Facebook Developer Console → Your App → **Webhooks** → **Add Callback URL**
2. **Callback URL:** `https://api.yourdomain.com/api/v1/webhooks/facebook`
3. **Verify Token:** (the value of `FACEBOOK_WEBHOOK_VERIFY_TOKEN` from `.env.prod`)
4. Subscribe to: `messages`, `messaging_postbacks`, `messaging_optins`, `message_deliveries`, `message_reads`

---

| Field | Value |
|---|---|
| **What it is** | Same as `FACEBOOK_APP_ID` — used by the frontend JavaScript SDK |
| **Value** | Same value as `FACEBOOK_APP_ID` |

### B13–B18. SMTP Email Settings ⚡ Optional but recommended

Required if you want email features (password reset, notifications, etc.).

| Variable | What it is | Example |
|---|---|---|
| `SMTP_HOST` | Outgoing mail server hostname | `smtp.gmail.com` / `smtp.sendgrid.net` |
| `SMTP_PORT` | SMTP port | `587` (TLS) or `465` (SSL) |
| `SMTP_SECURE` | Use SSL | `false` for port 587, `true` for 465 |
| `SMTP_USER` | SMTP username / email address | `noreply@yourdomain.com` |
| `SMTP_PASS` | SMTP password or app password | Your email password or app-specific password |
| `SMTP_FROM` | Sender address shown in emails | `noreply@yourdomain.com` |

**Common SMTP providers:**

| Provider | SMTP Host | Port | Notes |
|---|---|---|---|
| **Gmail** | `smtp.gmail.com` | `587` | Requires [App Password](https://myaccount.google.com/apppasswords), not your Gmail password |
| **SendGrid** | `smtp.sendgrid.net` | `587` | Use API key as `SMTP_PASS`, `apikey` as `SMTP_USER` |
| **Mailgun** | `smtp.mailgun.org` | `587` | Use SMTP credentials from Mailgun dashboard |
| **Resend** | `smtp.resend.com` | `465` | Use API key as password |
| **Brevo (Sendinblue)** | `smtp-relay.brevo.com` | `587` | Free tier available |

---

### B19. `SENTRY_DSN` ⚡ Optional — Error monitoring

| Field | Value |
|---|---|
| **What it is** | Sentry Data Source Name — enables error tracking and alerting |
| **How to get it** | [sentry.io](https://sentry.io) → Create Project → Node.js → DSN shown in setup |

## Part C — Domain & DNS

### C1. Domain Name — 🖊️ You provide this

You need a domain name pointing to your VPS IP address.

| DNS Record | Type | Name | Value |
|---|---|---|---|
| Frontend | `A` | `app` (→ `app.yourdomain.com`) | Your VPS IP address |
| Backend API | `A` | `api` (→ `api.yourdomain.com`) | Your VPS IP address |

**Alternative: single domain with path-based routing**

| DNS Record | Type | Name | Value |
|---|---|---|---|
| Everything | `A` | `@` (→ `yourdomain.com`) | Your VPS IP address |

Then in Nginx: `yourdomain.com` → frontend, `yourdomain.com/api` → backend.

**Registrars:** Namecheap, Cloudflare Registrar, Google Domains, GoDaddy, etc.
DNS propagation takes 1–24 hours after updating records.

## Quick Reference — Commands You'll Run

```bash
# ── On your LOCAL machine ─────────────────────────────────────

# 1. Generate a deploy SSH key pair
ssh-keygen -t ed25519 -C "messagesender-deploy" -f ~/.ssh/messagesender_deploy

# 2. Copy public key to VPS
ssh-copy-id -i ~/.ssh/messagesender_deploy.pub deploy@YOUR_VPS_IP

# 3. Print private key to paste into GitHub Secret VPS_SSH_KEY
cat ~/.ssh/messagesender_deploy


# ── On your VPS (as root, first time only) ────────────────────

# 4. Bootstrap VPS (installs Docker, Nginx, fail2ban, creates deploy user, etc.)
curl -fsSL https://raw.githubusercontent.com/ADNANKHALID4356/MessageSender/main/scripts/vps-setup-ubuntu.sh | sudo bash


# ── On your VPS (as deploy user) ─────────────────────────────

# 5. Run first deployment (clones repo, creates .env.prod, builds images)
bash <(curl -fsSL https://raw.githubusercontent.com/ADNANKHALID4356/MessageSender/main/scripts/vps-deploy.sh) \
  https://github.com/ADNANKHALID4356/MessageSender.git

# 6. After the above creates .env.prod, generate and fill in secrets
cd /opt/messagesender
bash scripts/generate-secrets.sh --write  # fills all auto-generated secrets
vim .env.prod                              # fill in URLs, Facebook, SMTP

# 7. Re-run the deploy (now that .env.prod is filled)
bash scripts/vps-deploy.sh https://github.com/ADNANKHALID4356/MessageSender.git


# ── On GitHub ─────────────────────────────────────────────────

# 8. Add GitHub Secrets for automated deploy on push:
#    Settings → Secrets → Actions → New repository secret
#    Add: VPS_HOST, VPS_USER, VPS_SSH_KEY (and optionally VPS_PORT, VPS_APP_DIR, FRONTEND_URL)
```

## Security Reminders

> - **Never share credentials** with anyone, including AI assistants, via chat, email, or any third-party system.
> - **Never commit `.env.prod`** to git (it's already in `.gitignore`).
> - **Back up `ENCRYPTION_KEY`** in a password manager immediately after first deploy — it cannot be recovered if lost.
> - **Rotate SSH keys** periodically and revoke the deploy key if you suspect it was compromised (GitHub Settings → Deploy keys).
> - Store all secrets in a password manager (1Password, Bitwarden, KeePass, etc.) immediately after generating them.
